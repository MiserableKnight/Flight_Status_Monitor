# 调度模式使用指南

## 快速开始

### 1. 运行测试（验证系统）
```bash
python test_scheduler.py
```
**预期结果**: 所有测试通过 ✅

### 2. 运行交互式模拟（查看运行过程）
```bash
python simulate_scheduler.py
```
**操作**: 按 Enter 键逐步查看一整天的调度运行过程

### 3. 运行真实调度模式
```bash
run_system.bat
```
选择选项 `1` - 调度模式

## 监控逻辑说明

### 监控策略

系统专注于航段数据页面监控，每分钟检查一次：
- 持续监控航段状态变化
- 自动检测起飞、降落事件
- 检测备降等异常情况
- 发送邮件通知

### 典型时间线（VJ105为例）

```
07:45 ✅ 计划起飞时间 → 开始监控
07:50 ✅ 滑出 (OUT) → 检测到状态变化
08:00 ✅ 起飞 (OFF) → 检测到起飞，发送通知
08:00-09:40 🔄 在空中 → 持续监控
09:45 ✅ 落地 (ON) → 检测到降落，发送通知
09:50 ✅ 滑入 (IN) → 航班完成
```

## 时间说明

### 内部时间（北京时间）
- 配置文件：`config/flight_schedule.py`
- 数据存储：`data/leg_data.csv`
- 调度判断：所有时间比较

### 展示时间（越南时间）
- 邮件通知：自动转换（北京时间-1小时）
- 用户看到：越南本地时间

### 转换规则
```
越南时间 06:45 → 北京时间 07:45
越南时间 = 北京时间 - 1小时
```

## 故障排查

### 如果没有收到邮件通知
**检查**:
1. `config.ini` 中 gmail 配置是否正确
2. 网络连接是否正常
3. SMTP 服务是否可用
4. 是否检测到状态变化

### 如果系统无法检测到状态变化
**检查**:
1. 航段数据是否正常抓取
2. `data/leg_data.csv` 文件是否存在
3. 查看日志文件了解详细错误信息

## 配置修改

### 修改航班时间
编辑 `config/flight_schedule.py`:
```python
'VJ105': {
    'scheduled_departure': '07:45',  # 北京时间 (越南时间+1)
    ...
}
```

### 修改运行时间
编辑 `config/config.ini`:
```ini
[scheduler]
start_time = 06:30  # 系统启动时间
end_time = 21:00    # 系统停止时间
```

## 性能指标

- **检查频率**: 每1分钟检查一次航班状态
- **页面切换**: 根据航班状态自动切换
- **资源占用**: 低（大部分时间只监控一个页面）
- **通知及时性**: 状态变化后1分钟内发送邮件
